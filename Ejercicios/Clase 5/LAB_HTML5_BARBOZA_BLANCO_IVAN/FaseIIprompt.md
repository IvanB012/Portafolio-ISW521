## Primer prompt:

Actúa como un Desarrollador Frontend Senior experto en estándares W3C y accesibilidad web. Tu tarea exclusiva es refactorizar por completo el código deficiente de la "Feria Nacional de Innovación Estudiantil" que te proporciono al final.
Debes aplicar de forma obligatoria las siguientes correcciones estructurales:
1. Estructura Semántica HTML5: Cambia las clases genéricas de maquetación por elementos nativos: `<header>` para el bloque del título principal, `<nav>` para el menú de enlaces, `<main>` envolviendo el núcleo del contenido único, `<section>` para los apartados principales (#inicio, #proyectos, #agenda, etc.) y `<article>` para aislar de forma independiente cada tarjeta de los proyectos finalistas.
2. Accesibilidad en Formulario: Reemplaza los divs de texto plano por etiquetas `<label>`. Enlázalas de manera estricta usando el atributo 'for' apuntando directamente al 'id' de su respectivo `<input>`, `<select>` o `<textarea>`.
3. Agrupación Lógica: Estructura el formulario dividiéndolo en dos grandes bloques mediante la etiqueta `<fieldset>`, definiendo sus títulos contextuales con `<legend>` (por ejemplo: "Datos del Equipo" y "Detalles del Proyecto").
4. Componente Predictivo: Transforma el `<select>` del campo "Carrera o programa académico" en un `<input type="text">` dinámico vinculado a un elemento `<datalist>` que contenga todas las opciones originales de carreras académicas.
5. Arquitectura de Tabla: Refactoriza la tabla de horarios añadiendo explícitamente las etiquetas `<caption>` (con un título accesible), `<thead>` (encabezados de días), `<tbody>` (horas y actividades) y `<tfoot>` (para la fila de totales). Asegúrate de incluir el atributo scope="col" o scope="row" en todas las celdas `<th>`.
6. Optimización de Video: Modifica la etiqueta `<video>`. Incorporá el atributo 'poster' con una imagen de previsualización ("img/poster_feria.jpg"), el atributo 'controls' para permitir reproducir el video, y dos etiquetas `<source>` internas para dar soporte tanto a formato video/mp4 como video/webm.
7. Seguridad en Iframes: Modifica el <iframe> de Google Maps agregando loading="lazy" para rendimiento, title="..." con la descripción textual accesible del mapa, y aislamiento perimetral estricto mediante sandbox="allow-scripts allow-same-origin".
8. Corrección de Anidamiento Válido: Elimina los elementos de bloque (divs y h3) que están rotos dentro del elemento inline `<span>` en la sección de contacto, y dentro de los enlaces `<a>` en las redes sociales.
Restricciones: Mantén intacto el bloque de estilos CSS del <head> (solo adapta los selectores necesarios si eliminaste divs de clases principales). No inventes textos ni borres datos originales del evento. Entrega ÚNICAMENTE el bloque de código HTML5 limpio y refactorizado dentro de un bloque de código, sin comentarios extras.
